# Гайд-онбординг: Руководитель

Роль **Руководитель** имеет полный доступ ко всем разделам системы. Вы управляете командой, контролируете проекты и договоры всех менеджеров, настраиваете видимость для производства, просматриваете сводную аналитику и ведёте учёт ресурсов.

---

## Меню системы

После входа вы увидите в боковом меню **все** разделы:

| Раздел | Описание |
|--------|---------|
| Главная | Дашборд с приветствием |
| Клиенты | Воронка продаж всех менеджеров |
| Договоры | Все договоры |
| Календарь | Годовой вид мероприятий + создание событий |
| Проекты | Все проекты |
| Производство | Обзор задач и закупок по проектам |
| Дизайнер | Все макеты дизайнеров |
| Менеджер | Задачи и расходы менеджеров |
| Инвентарь | Склад: реквизит и ткани |
| Инфо | Персонал и подрядчики |
| Почта | Входящие заявки с сайта |
| Статистика | Аналитика по воронке и выручке |
| История | Журнал аудита всех действий |
| Сообщения | Общий командный чат |
| Настройки | Пользователи, открытые месяцы, профиль |

---

## 1. Вход в систему

1. Откройте браузер и перейдите по адресу системы (например, `https://crm.maketdecor.ru`)
2. Введите логин и пароль, выданные администратором
3. Нажмите **"Войти"**

> Если вы забыли пароль — обратитесь к системному администратору. Самостоятельный сброс пароля не предусмотрен.

---

## 2. Главная (Дашборд)

После входа вы попадаете на дашборд с приветствием и вашим именем.

- **Уведомления** (колокольчик в шапке) — оповещения о новых заявках, изменениях статусов, сообщениях. Можно отметить прочитанными по одному или все сразу.
- **Смена обоев** — кнопка в шапке, выбор из 15 вариантов фона. Сохраняется для вашего аккаунта.
- **Глобальный поиск** — поле поиска в шапке. Ищет по клиентам (имя, площадка, источник), договорам (имя, площадка, номер счёта), проектам (площадка, описание). Минимум 2 символа, до 5 результатов по каждой категории.

---

## 3. Управление пользователями

Раздел **"Настройки"** — только для руководителя.

### Создание пользователя

1. Перейдите в **Настройки**
2. Нажмите **"Добавить пользователя"**
3. Заполните поля:
   - **Имя** — отображаемое имя
   - **Логин** — используется для входа
   - **Пароль** — временный пароль (сообщите сотруднику)
   - **Роль** — `Менеджер`, `Производство` или `Дизайнер`
4. Нажмите **"Сохранить"**

### Редактирование пользователя

1. В списке пользователей нажмите иконку редактирования
2. Измените нужные поля (имя, пароль, роль, статус активности)
3. Сохраните изменения

### Флаг "Доступ к разделу Инфо"

По умолчанию менеджеры не видят разделы "Персонал" и "Подрядчики". Чтобы открыть доступ:

1. В редактировании пользователя найдите переключатель **"Доступ к разделу Инфо"**
2. Активируйте его и сохраните

### Деактивация пользователя

Удаление деактивирует аккаунт (флаг `isActive`). Данные (клиенты, проекты) сохраняются.

### Профиль

В разделе **"Настройки"** вы также можете изменить своё имя и пароль.

---

## 4. Клиенты

Раздел **"Клиенты"** — воронка продаж. Вы видите клиентов **всех** менеджеров.

### Воронка продаж

| Статус | Значение |
|--------|---------|
| `Встреча` | Назначена встреча |
| `Обсуждение` | Обсуждение деталей |
| `Смета` | Подготовка сметы |
| `Договор` | Договор заключён |
| `Отказ` | Клиент отказался |

### Просмотр и фильтрация

- Таблица с сортировкой по дате, менеджеру, статусу, типу мероприятия
- Вкладки: **Активные клиенты** / **Отказы**
- Фильтр по менеджеру (только для руководителя)
- Типы мероприятий: Свадьба, Корпоратив, День рождения, Другое

### Карточка клиента

Нажмите на строку — откроется карточка:
- Контактная информация, даты (обращения, встречи, мероприятия)
- Площадка, тип мероприятия, описание идеи, источник обращения
- Прикреплённые сметы (файлы с версиями и командными версиями)

### Конвертация в договор

1. Откройте карточку клиента → **"Конвертировать в договор"**
2. Система автоматически создаёт договор, переносит сметы, обновляет статус на `Договор`

### Отклонение клиента

1. Нажмите **"Отклонить"** → укажите причину → подтвердите
2. Статус меняется на `Отказ`, запись сохраняется для статистики

---

## 5. Договоры

Раздел **"Договоры"** — вы видите договоры **всех** менеджеров.

### Карточка договора

- Данные клиента, даты мероприятия и подписания
- Сумма договора, предоплата, номер счёта
- **Статус макета** — отслеживание работы дизайнера:

| Статус макета | Значение |
|---------------|---------|
| `Ожидает` | Ожидает начала работы |
| `В работе` | Дизайнер работает |
| `Согласован` | Макет одобрен клиентом |
| `Передан` | Передан в производство |
| `Отменён` | Отменён |

- **Файлы**: договор (Word/PDF), счёт
- **Референсные изображения**: фото-вдохновения от клиента для дизайнера
- Связь с исходным клиентом и проектом

---

## 6. Проекты

Раздел **"Проекты"** — вы видите проекты **всех** менеджеров.

### Список проектов

- Карточки проектов, сгруппированные по месяцам
- Цветовые метки, прогресс задач, индикатор завершения
- Фильтр по месяцу

### Карточка проекта

- Дата и место мероприятия, описание, цвет в календаре
- Связь с договором
- **Задачи производства** — чеклист с отметками выполнения (кто и когда)
- **Закупки** — список материалов с отметками о покупке
- **Чат проекта** — общение команды в реальном времени (Socket.io)
- **Фотографии** — эскизы заказа и фото производства

---

## 7. Производство (обзор)

Раздел **"Производство"** — сводный обзор всех проектов с задачами и закупками.

Для каждого проекта отображается:
- Название, дата, менеджер
- Прогресс задач (полоса прогресса + чекбоксы)
- Прогресс закупок (полоса прогресса + чекбоксы)

Используйте этот раздел для быстрого контроля готовности без перехода в каждый проект.

---

## 8. Календарь

Раздел **"Календарь"** — годовой вид мероприятий.

### Просмотр

- Сетка по неделям (Пн-Вс), навигация по годам
- Проекты отображаются цветными метками:
  - Оранжевый — монтаж
  - Синий — демонтаж
  - Зелёный — бронь
- Нажмите на метку — переход к проекту

### Пользовательские события (только руководитель)

1. Нажмите **"Добавить событие"**
2. Укажите дату, название, цвет
3. Сохраните — событие появится в календаре
4. Для удаления — кнопка удаления рядом с событием

---

## 9. Открытие месяцев для производства

**Ключевая функция руководителя**: производство видит проекты **только** в месяцах, которые вы открыли.

### Как открыть месяц

1. Перейдите в **"Настройки"**
2. Найдите блок **"Открытые месяцы для производства"**
3. Выберите год и нажмите на нужные месяцы
4. Нажмите **"Сохранить"**

После этого сотрудники с ролью "Производство" увидят проекты выбранных месяцев.

---

## 10. Макеты дизайнера

Раздел **"Дизайнер"** — все макеты от всех дизайнеров.

- Карточки макетов: номер, месяц, даты (начала, монтажа), сложность (1-5 звёзд), количество дней, статус, превью изображения
- Фильтр по месяцу
- Загруженные изображения по зонам (церемония, президиум, банкетный зал, фотозона, вход и др.)

---

## 11. Менеджер (задачи и расходы)

Раздел **"Менеджер"** — вы видите задачи и расходы **всех** менеджеров.

### Личные задачи менеджеров

- Список задач: описание, срок, статус выполнения
- Можно видеть, кто назначил задачу

### Расходы

- Категории: общие, расходники, проектные
- Сумма, описание, дата, привязка к проекту

---

## 12. Склад: инвентарь и ткани

### Инвентарь

Раздел **"Инвентарь"** — иерархический каталог реквизита.

- **Дерево категорий**: создание, вложенность до 3 уровней
- **Позиции**: название, цвет, количество, фото
- **Журнал повреждений**: фиксация порчи с описанием, датой, количеством
- **Поиск** по названию и цвету
- Добавление/редактирование категорий и позиций

### Ткани

Раздел **"Ткани"** (ссылка из раздела "Инвентарь"):

- Материал, цвет, ширина (см), раскрой, общая длина (м)
- Поставщик, год покупки, примечания
- Группировка по типу материала

---

## 13. Персонал и подрядчики

Раздел **"Инфо"** — справочники сотрудников и поставщиков.

### Персонал

Категории:
- **Основная команда** — штатные сотрудники
- **Фрилансы (парни)** — мужчины-фрилансеры
- **Фрилансы (девочки)** — женщины-фрилансеры
- **Газелисты** — водители

Для каждого: ФИО, должность, паспорт, дата рождения, адрес, телефон, наличие авто, Telegram, примечания.

### Подрядчики

Справочник поставщиков по категориям: компания, адрес, телефон, Telegram, примечания, кто добавил.

---

## 14. Почта (входящие заявки)

Раздел **"Почта"** — заявки, поступившие через форму на сайте.

- Список заявок: отправитель, тема, текст, дата, статус (прочитано/не прочитано)
- Счётчик непрочитанных заявок
- Можно отметить как прочитанное, назначить ответственного
- Из заявки можно создать клиента в воронке

---

## 15. Статистика

Раздел **"Статистика"** — сводные данные по **всем** менеджерам.

| Метрика | Описание |
|---------|---------|
| Всего клиентов | Общее количество активных клиентов |
| Всего договоров | Количество подписанных договоров |
| Всего проектов | Количество проектов |
| Отказы | Количество отклонённых клиентов |
| Конверсия | % клиентов, дошедших до договора |
| Воронка продаж | Распределение по статусам с прогресс-барами |
| Выручка по месяцам | Суммы договоров за выбранный год |

---

## 16. История изменений

Раздел **"История"** — полный журнал аудита. Вы видите действия **всех** пользователей.

- Кто, что, когда изменил
- Тип сущности: клиент, договор, проект, задача, инвентарь, ткань, персонал, подрядчик, пользователь, файл, настройки
- Фильтр по типу сущности
- Пагинация (до 200 записей на страницу)

---

## 17. Общий чат

Раздел **"Сообщения"** — командный мессенджер в реальном времени.

- Общий канал для всей команды (все роли)
- Сообщения доставляются мгновенно (Socket.io)
- История сохраняется (последние 100 сообщений)
- Прикрепление файлов

**Отдельно**: в каждом проекте есть свой чат — для обсуждения конкретного мероприятия.

## Гайд-онбординг: первые шаги в системе

Пошаговое руководство для нового руководителя — от первого входа до полноценной работы со всеми разделами.

### Шаг 1. Первый вход и настройка

1. Откройте `https://crm.maketdecor.ru`, введите логин и пароль, нажмите **"Войти"**
2. Вы попали на **Дашборд**. Слева — боковое меню со всеми 15 разделами
3. Нажмите на колокольчик в шапке — проверьте уведомления
4. Смените обои дашборда на понравившиеся (кнопка в шапке)
5. Перейдите в **"Настройки"** → измените пароль на свой в разделе профиля

### Шаг 2. Создайте команду

1. В **"Настройках"** перейдите к блоку **"Пользователи"**
2. Нажмите **"Добавить пользователя"** → создайте аккаунты для менеджеров (роль "Менеджер")
3. Создайте аккаунт для дизайнера (роль "Дизайнер")
4. Создайте аккаунты для производства (роль "Производство")
5. Для менеджеров, которым нужен доступ к справочникам сотрудников — включите флаг **"Доступ к разделу Инфо"**
6. Сообщите каждому сотруднику логин и временный пароль

### Шаг 3. Наполните справочники

1. Перейдите в **"Инвентарь"** → создайте дерево категорий (Арки, Вазы, Подсвечники, Текстиль, Оборудование...)
2. Добавьте позиции инвентаря в каждую категорию (название, цвет, количество)
3. Перейдите в **"Ткани"** → добавьте текстиль (материал, цвет, ширина, поставщик)
4. Откройте **"Инфо → Персонал"** → внесите штатных сотрудников, фрилансеров и водителей
5. Откройте **"Инфо → Подрядчики"** → добавьте поставщиков по категориям

### Шаг 4. Откройте месяцы для производства

1. В **"Настройках"** найдите блок **"Открытые месяцы"**
2. Выберите текущий и следующий месяцы, нажмите **"Сохранить"**
3. Теперь производство будет видеть проекты в этих месяцах

### Шаг 5. Освойте воронку продаж

1. Откройте **"Клиенты"** — здесь будут появляться клиенты менеджеров
2. Попробуйте фильтры: по статусу, менеджеру, типу мероприятия
3. Переключитесь между вкладками "Активные" и "Отказы"
4. Откройте любую карточку клиента — изучите все поля и вкладки (сметы, история)

### Шаг 6. Познакомьтесь с договорами и проектами

1. **"Договоры"** — здесь появляются записи после конвертации клиентов. Изучите карточку: файлы, референсы, статус макета
2. **"Проекты"** — карточки мероприятий с задачами, закупками, чатом и фото
3. **"Производство"** — сводный обзор всех проектов с прогрессом задач и закупок

### Шаг 7. Изучите календарь

1. Откройте **"Календарь"** — годовой вид с цветными метками
2. Легенда: оранжевый — монтаж, синий — демонтаж, зелёный — бронь
3. Попробуйте добавить пользовательское событие (нерабочий день, корпоратив)
4. Нажмите на метку проекта — перейдёте в его карточку

### Шаг 8. Проверьте работу дизайнера

1. Откройте **"Дизайнер"** — список всех макетов
2. Изучите карточку макета: даты, сложность, статус, изображения по зонам
3. Следите за сменой статусов: В работе → На согласовании → Согласован → Передан

### Шаг 9. Настройте коммуникацию

1. Откройте **"Сообщения"** — напишите приветственное сообщение команде
2. Сообщения доставляются в реальном времени всем сотрудникам
3. Для обсуждения конкретного мероприятия используйте чат внутри проекта

### Шаг 10. Проверьте входящие заявки

1. Откройте **"Почта"** — здесь появляются заявки с сайта
2. Непрочитанные заявки выделены отдельной иконкой
3. Из заявки можно создать клиента в воронке

### Шаг 11. Изучите аналитику

1. Откройте **"Статистика"** — воронка, конверсия, выручка по месяцам
2. Данные отображаются по всем менеджерам
3. Используйте фильтр по году

### Шаг 12. Аудит и контроль

1. **"История"** — полный журнал всех действий всех пользователей
2. Фильтруйте по типу сущности (клиент, договор, проект и т.д.)
3. Используйте для расследования спорных ситуаций

---

## Ежедневный маршрут руководителя

> **Утро**: Уведомления → Сообщения → Почта (новые заявки) → Статистика (воронка)
>
> **День**: Клиенты/Договоры (контроль) → Проекты/Производство (прогресс) → Календарь (планирование) → Дизайнер (статусы макетов) → Инвентарь (остатки)
>
> **По необходимости**: Настройки (пользователи, открытие месяцев) → История (расследование)

## Типичные рабочие сценарии

### Сценарий 1: Начало нового месяца

1. **Настройки → Открытые месяцы** → откройте нужный месяц
2. Производство увидит проекты этого месяца

### Сценарий 2: Контроль менеджеров

1. **Статистика** — воронка и конверсия по всем менеджерам
2. **Клиенты** — фильтр по менеджеру, проверка "застрявших" клиентов
3. **История** — проверка активности сотрудника за период

### Сценарий 3: Новый сотрудник

1. **Настройки → Пользователи → Добавить** → задайте роль и пароль
2. Сообщите логин и пароль сотруднику
3. При необходимости — включите **"Доступ к разделу Инфо"**

### Сценарий 4: Расследование инцидента

1. **История** → фильтр по пользователю и типу сущности
2. Найдите нужное действие, просмотрите детали
