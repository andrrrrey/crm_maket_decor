# Инструкция: Руководитель

Роль **Руководитель** имеет полный доступ ко всем разделам системы. Руководитель управляет командой, контролирует проекты и договоры всех менеджеров, настраивает видимость для производства и дизайнеров, просматривает сводную аналитику.

## 1. Вход в систему

1. Откройте браузер и перейдите по адресу системы (например, `https://crm.maketdecor.ru`)
2. Введите логин и пароль, выданные администратором
3. Нажмите **«Войти»**

После успешного входа вы попадёте на главную страницу (Дашборд). В левой части экрана — боковое меню со всеми доступными разделами.

> Если вы забыли пароль — обратитесь к системному администратору. Самостоятельный сброс пароля через форму не предусмотрен.

---

## 2. Главная (Дашборд)

Дашборд — ваша личная рабочая область при входе в систему.

**Что здесь есть:**
- Фоновое изображение (можно сменить — кнопка в правом верхнем углу, выбор из 15 вариантов)
- Быстрые ссылки на ключевые разделы
- Уведомления (колокольчик в шапке)

**Смена обоев:**
1. Нажмите иконку настроек / обоев в шапке или на самом дашборде
2. Выберите понравившееся изображение из галереи
3. Изменение применяется мгновенно и сохраняется для вашего аккаунта

## 3. Управление пользователями

Раздел **«Настройки → Пользователи»** — только для руководителя.

### Создание пользователя

1. Перейдите в **Настройки**
2. Нажмите **«Добавить пользователя»**
3. Заполните поля:
   - **Имя** — отображаемое имя
   - **Email / Логин** — используется для входа
   - **Пароль** — временный пароль (сообщите сотруднику)
   - **Роль** — выберите из списка: `Менеджер`, `Производство`, `Дизайнер`
4. Нажмите **«Сохранить»**

### Редактирование пользователя

1. В списке пользователей нажмите иконку редактирования напротив нужного
2. Измените нужные поля (имя, пароль, роль)
3. Сохраните изменения

### Флаг «Доступ к разделу Инфо»

По умолчанию менеджеры не видят разделы «Персонал» и «Подрядчики». Чтобы открыть доступ:

1. В редактировании пользователя найдите переключатель **«Доступ к разделу Инфо»**
2. Активируйте его
3. Сохраните

После этого менеджер увидит в меню разделы «Персонал» и «Подрядчики».

### Удаление пользователя

Удаление деактивирует аккаунт. Данные (клиенты, проекты) сохраняются и остаются привязаны к этому пользователю.

## 4. Клиенты

Раздел **«Клиенты»** — воронка продаж. Руководитель видит клиентов **всех менеджеров**.

### Воронка продаж

Статусы клиента (слева → вправо по воронке):

| Статус | Значение |
|--------|---------|
| `MEETING` | Назначена встреча |
| `DISCUSSION` | Обсуждение деталей |
| `ESTIMATE` | Подготовка сметы |
| `CONTRACT` | Договор заключён |
| `REJECTED` | Отказ |

### Просмотр клиентов

- Список отображается в виде таблицы
- Доступна сортировка по дате, менеджеру, статусу, типу мероприятия
- Фильтры: по статусу, менеджеру, типу мероприятия, дате

### Карточка клиента

Нажмите на строку, чтобы открыть детальную карточку. В ней:
- Контактная информация клиента
- Дата обращения, дата встречи, дата мероприятия
- Площадка, тип мероприятия, описание идеи
- Прикреплённые сметы (файлы)
- История изменений по этому клиенту

### Конвертация в договор

Когда клиент соглашается на условия:

1. Откройте карточку клиента
2. Нажмите **«Конвертировать в договор»**
3. Подтвердите действие

Система автоматически:
- Создаёт запись в разделе «Договоры» с копией данных клиента
- Переносит прикреплённые сметы
- Обновляет статус клиента на `CONTRACT`

### Отклонение клиента

1. Откройте карточку клиента
2. Нажмите **«Отклонить»**
3. Укажите причину отказа
4. Нажмите **«Подтвердить»**

Статус меняется на `REJECTED`. Запись сохраняется для статистики.

## 5. Договоры

Раздел **«Договоры»** — руководитель видит договоры **всех менеджеров**.

### Карточка договора

Содержит:
- Данные клиента (перенесены при конвертации)
- Даты мероприятия и подписания
- Сумму договора и условия предоплаты
- Номер счёта
- **Статус макета** (согласование с дизайнером):
  - `PENDING` — ожидает
  - `IN_PROGRESS` — в работе
  - `APPROVED` — согласован
  - `TRANSFERRED` — передан в производство
  - `CANCELLED` — отменён
- Прикреплённые файлы: договор (Word/PDF), счёт
- Изображения-референсы для дизайнера

### Отслеживание статуса макетов

Руководитель видит сводную картину по всем договорам: на каком этапе согласования находится каждый макет. Это помогает контролировать дизайнеров.

---

## 6. Проекты и производство

Раздел **«Проекты»** — руководитель видит проекты **всех менеджеров**.

### Карточка проекта

- Дата и место мероприятия
- Описание
- Цвет в календаре (цветовое кодирование)
- **Задачи производства** — чеклист выполнения (кто и когда отметил)
- **Список закупок** — материалы для заказа
- **Чат проекта** — общение команды по конкретному проекту
- **Фотографии** — эскизы заказа и фото с производства

### Контроль выполнения задач

На уровне проекта видно:
- Какие задачи выполнены / не выполнены
- Кто отметил задачу выполненной и когда

Это позволяет руководителю контролировать готовность без личного присутствия.

## 7. Календарь — открытие месяцев

**Ключевая функция руководителя**: контроль того, какие месяцы видит команда производства.

### Зачем это нужно

Сотрудники производства **не видят** проекты автоматически. Руководитель открывает конкретные месяцы, и только тогда производство видит проекты в этих месяцах.

### Как открыть месяц для производства

1. Перейдите в раздел **«Настройки»**
2. Найдите блок **«Открытые месяцы для производства»**
3. Выберите год и нажмите на месяц/месяцы, которые нужно открыть
4. Нажмите **«Сохранить»**

После этого сотрудники с ролью «Производство» увидят проекты в выбранных месяцах.

### Просмотр календаря

Раздел **«Календарь»**:
- Годовой вид: сетка по неделям (Пн–Вс)
- Проекты отображаются цветными метками в ячейках дат
- Каждый менеджер — свой цвет
- Наведение на метку — всплывающая подсказка с деталями проекта

## 8. Макеты дизайнера

Раздел **«Дизайнер»** — руководитель видит все макеты.

### Что здесь отображается

- Список всех макетов от всех дизайнеров
- Для каждого: месяц, даты монтажа, сложность (1–5), статус
- Загруженные изображения по зонам (церемония, президиум и т.д.)

### Отслеживание работы дизайнеров

По статусам макетов руководитель видит загруженность дизайнеров и степень готовности каждого проекта.

---

## 9. Склад: инвентарь и ткани

### Инвентарь

Раздел **«Инвентарь»** — иерархический каталог реквизита.

**Категории:**
- Дерево категорий (добавить / переименовать / удалить категорию)
- Вложенность произвольная

**Позиции инвентаря:**
- Название, количество, фото, статус
- Журнал повреждений — фиксация порчи с датой и описанием

**Действия руководителя:**
1. Создать / изменить категорию — кнопка в боковой панели дерева
2. Добавить позицию — кнопка «Добавить» внутри категории
3. Зафиксировать повреждение — открыть позицию → кнопка «Повреждение» → описание + дата

### Ткани

Раздел **«Ткани»** — отдельный учёт текстиля.

- Название, артикул, цвет, количество (метры/единицы)
- Добавление, редактирование, удаление позиций

## 10. Персонал и подрядчики

Разделы доступны руководителю и менеджерам с флагом `hasInfoAccess`.

### Персонал

Раздел **«Персонал»** — база сотрудников и фрилансеров.

Категории:
- `CORE_TEAM` — штатные сотрудники
- `FREELANCE_MALE` — мужчины-фрилансеры
- `FREELANCE_FEMALE` — женщины-фрилансеры
- `DRIVERS` — водители

Для каждой позиции: имя, контакты, роль, примечания.

### Подрядчики

Раздел **«Подрядчики»** — справочник поставщиков по категориям (цветы, аренда, транспорт и т.д.).

---

## 11. Входящие заявки с сайта

Раздел **«Входящие»** — заявки, поступившие через форму на сайте.

- Система периодически проверяет почту (IMAP) и создаёт записи автоматически
- Каждая запись содержит: имя, телефон, email, текст заявки, дату
- Руководитель и менеджеры видят все заявки
- Из заявки можно создать клиента в воронке (кнопка «Создать клиента»)

## 12. Аналитика и статистика

Раздел **«Статистика»** — сводные данные по работе студии.

**Доступные отчёты:**

| Отчёт | Что показывает |
|-------|----------------|
| Воронка продаж | Количество клиентов на каждом этапе воронки |
| Конверсия | % перехода между этапами |
| Выручка по месяцам | Суммы договоров, разбивка по периодам |
| Активность менеджеров | Количество клиентов и договоров по каждому менеджеру |

Руководитель видит данные по **всем менеджерам**. Менеджер видит только свои данные.

---

## 13. История изменений

Раздел **«История»** — полный журнал аудита.

Руководитель видит все действия всех пользователей:
- Кто, что, когда изменил
- Тип действия: создание, редактирование, удаление, завершение задачи и т.д.
- Детали изменения: было → стало

**Фильтры:**
- По пользователю
- По типу сущности (клиент, договор, проект и т.д.)
- По дате

Используйте историю для расследования спорных ситуаций и контроля работы команды.

---

## 14. Общий чат

Раздел **«Сообщения»** — командный мессенджер в реальном времени.

- Общий канал для всей команды (все роли)
- Сообщения отображаются в реальном времени (Socket.io)
- История сообщений сохраняется

**Отдельно**: в каждом проекте есть свой чат — для обсуждения конкретного мероприятия.

## Пример пути пользователя в платформе (CJM)

Сквозной пример действий руководителя Сергея после авторизации — обычный рабочий день, в котором задействованы все ключевые разделы платформы.

### Шаг 0. Авторизация
1. Сергей открывает `https://crm.maketdecor.ru`, вводит логин/пароль, нажимает **«Войти»**.
2. Попадает на **Дашборд**. В боковом меню — все разделы без ограничений: Дашборд, Клиенты, Договоры, Проекты, Календарь, Дизайнер, Производство, Склад, Инфо, Входящие, Сообщения, Статистика, История, Настройки.
3. В шапке справа — колокольчик с уведомлениями: 5 новых событий за ночь.

### Шаг 1. Утренний обзор
1. Сергей кликает по **колокольчику** — видит: новые заявки с сайта, изменения статусов клиентов, новые сообщения в чатах.
2. Открывает **«Дашборд»**, бегло смотрит ключевые показатели и быстрые ссылки.
3. Переходит в **«Сообщения»** → читает общий чат, отвечает на срочные вопросы команды.

### Шаг 2. Контроль воронки и менеджеров
1. Открывает **«Статистика»** → смотрит сводную **воронку продаж** по всем менеджерам.
2. Видит, что у одного менеджера много клиентов «застряли» в `DISCUSSION` дольше 14 дней.
3. Переходит в **«Клиенты»**, фильтрует по этому менеджеру и статусу `DISCUSSION`.
4. Открывает 2–3 проблемные карточки, смотрит историю переписки, ставит себе задачу обсудить это на планёрке.
5. Открывает **«История изменений»**, фильтрует по нужному пользователю за вчера — проверяет активность.

### Шаг 3. Обработка входящих заявок
1. Сергей открывает **«Входящие»** — видит 3 новые заявки.
2. Распределяет их: кому-то пишет в общий чат «@Анна, бери свадьбу 12.07», для крупного корпоратива создаёт клиента сам и назначает ответственного менеджера.

### Шаг 4. Контроль договоров и крупных проектов
1. Переходит в **«Договоры»**, сортирует по сумме — открывает топ-5 крупнейших активных договоров.
2. В каждой карточке проверяет статус макета (`PENDING`, `IN_PROGRESS`, `APPROVED`, `TRANSFERRED`).
3. По договору с задержкой статуса макета пишет в общий чат: «@дизайнер, что по арке для свадьбы 12.07?».

### Шаг 5. Управление производством и календарём
1. Открывает **«Настройки → Открытые месяцы»** — открывает август для производства, чтобы команда уже видела предстоящие проекты.
2. Заходит в **«Календарь»** — смотрит плотность мероприятий по неделям, замечает «горячую» неделю в июле, где 4 свадьбы подряд.
3. Открывает **«Проекты»**, проваливается в самый сложный проект недели:
   - Проверяет прогресс **задач** и **закупок**
   - Читает **чат проекта**, при необходимости пишет распоряжения
   - Смотрит загруженные **фотографии производства**
4. Заходит в **«Производство»** — общий обзор всех активных проектов в производстве.

### Шаг 6. Склад и ресурсы
1. Открывает **«Склад → Инвентарь»**, смотрит остатки ключевых позиций.
2. Замечает дефицит свечей — пишет в общий чат закупщику.
3. Заходит в **«Склад → Ткани»** — проверяет наличие нужного шифона на ближайшие свадьбы.
4. Открывает **«Инфо → Персонал»** и **«Инфо → Подрядчики»** — проверяет контакты монтажной бригады, при необходимости добавляет нового подрядчика.

### Шаг 7. Управление командой
1. К Сергею пришёл новый сотрудник. Он открывает **«Настройки → Пользователи» → «Добавить пользователя»**.
2. Заполняет имя, логин, временный пароль, выбирает роль (например, «Производство»), при необходимости включает доступ к разделу «Инфо».
3. Сохраняет — сотрудник может войти в систему.
4. Если сотрудник уволился — Сергей открывает его карточку, удаляет/деактивирует пользователя.

### Шаг 8. Аналитика и решения
1. В конце дня Сергей снова открывает **«Статистика»**:
   - **Конверсия** между этапами воронки
   - **Выручка по месяцам**
   - **Активность менеджеров** — кто сколько закрыл договоров
2. На основе данных формирует план на следующую неделю.

### Шаг 9. Расследование инцидента (по необходимости)
1. Менеджер пожаловался, что «исчез файл из договора».
2. Сергей открывает **«История изменений»**, фильтрует по сущности «договор» и нужной дате.
3. Видит, кто и когда удалил файл, делает скриншот, обсуждает с сотрудником.

> Сквозной маршрут руководителя: **Дашборд → Уведомления/Сообщения → Статистика → Клиенты/Договоры/Проекты → Календарь/Производство/Склад → Настройки (пользователи, открытые месяцы) → История**. Руководитель — единственная роль, которая использует все разделы платформы.

## Типичные рабочие сценарии

### Сценарий 1: Начало нового месяца

1. Перейдите в **Настройки → Открытые месяцы**
2. Откройте нужный месяц для производства
3. Сотрудники производства увидят проекты этого месяца

### Сценарий 2: Контроль работы менеджеров

1. Откройте **Статистику** — проверьте воронку и конверсию
2. Откройте **Клиенты** — отфильтруйте по менеджеру, посмотрите задержавшихся клиентов
3. В **Истории** проверьте активность за нужный период

### Сценарий 3: Добавление нового сотрудника

1. **Настройки → Пользователи → Добавить пользователя**
2. Выдайте логин и временный пароль сотруднику
3. При необходимости — включите флаг **«Доступ к разделу Инфо»**

### Сценарий 4: Расследование инцидента

1. Откройте **Историю изменений**
2. Отфильтруйте по пользователю и дате
3. Найдите нужное действие и просмотрите детали изменения
