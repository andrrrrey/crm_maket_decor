# Инструкция: Производство

Роль **Производство** — сотрудники, которые непосредственно изготавливают и монтируют декор. В системе они видят задачи по предстоящим мероприятиям, управляют складом и ведут журнал выполненных работ.

**Ключевое ограничение**: производство видит только проекты в **месяцах, открытых руководителем**. Пока руководитель не откроет месяц — проекты в нём не отображаются.

## 1. Вход в систему

1. Перейдите по адресу системы (например, `https://crm.maketdecor.ru`)
2. Введите логин и пароль
3. Нажмите **«Войти»**

В боковом меню отображаются только разделы, доступные роли «Производство». Часть разделов (клиенты, договоры, статистика) скрыта — они не нужны для производственной работы.

---

## 2. Главная (Дашборд)

После входа вы попадаете на дашборд.

**Что здесь есть:**
- Фоновое изображение (можно сменить)
- Уведомления о новых задачах и сообщениях (колокольчик в шапке)

**Смена фона дашборда:**
1. Нажмите иконку смены обоев
2. Выберите изображение из галереи

## 3. Проекты — просмотр и работа с задачами

Раздел **«Проекты»** — список мероприятий, которые нужно подготовить.

### Почему проекты могут не отображаться

Производство видит проекты только в **открытых месяцах**. Если список пуст:
- Руководитель ещё не открыл текущий месяц
- Уточните у руководителя, когда планируется открытие

### Список проектов

Каждый проект в списке показывает:
- Название мероприятия
- Дата и место проведения
- Прогресс задач (например, «5/12 выполнено»)
- Описание

### Открытие карточки проекта

Нажмите на строку проекта — откроется полная карточка с вкладками:
- **Задачи** — производственный чеклист
- **Закупки** — что нужно купить
- **Чат** — общение по проекту
- **Фотографии** — эскизы и фото производства

## 4. Выполнение производственных задач

Вкладка **«Задачи»** — основная рабочая область производства.

### Структура задач

Задачи создаёт менеджер или руководитель. Каждая задача — конкретный шаг производственного процесса:
- Изготовление конструкций
- Сборка элементов декора
- Подготовка реквизита
- Упаковка
- Доставка / монтаж

### Отметка задачи выполненной

1. Откройте проект → вкладка **«Задачи»**
2. Найдите нужную задачу
3. Нажмите на **чекбокс** слева от задачи
4. Задача отмечается как выполненная — система записывает **ваше имя** и **дату/время** выполнения

> Отмена выполнения: повторно нажмите на чекбокс, чтобы снять отметку.

### Просмотр деталей задачи

Нажмите на название задачи, чтобы увидеть полное описание (если оно добавлено менеджером).

### Контроль прогресса

Вверху вкладки «Задачи» отображается:
- Общее количество задач
- Сколько выполнено
- Прогресс-бар

## 5. Закупки проекта

Вкладка **«Закупки»** — список материалов, которые нужно приобрести.

### Работа со списком закупок

Список создаётся менеджером. Вы можете:

**Отметить позицию купленной:**
1. Найдите нужную позицию
2. Нажмите чекбокс
3. Позиция отмечается как купленная

**Просмотр деталей:**
- Название материала/расходника
- Количество и единицы измерения
- Примечания (если добавлены менеджером)

---

## 6. Чат проекта

Вкладка **«Чат»** — общение команды по конкретному мероприятию.

**Как пользоваться:**
1. Откройте проект → вкладка **«Чат»**
2. Введите сообщение в поле внизу
3. Нажмите **Enter** или кнопку отправки
4. Сообщение доставляется всем участникам в реальном времени

**Используйте чат проекта для:**
- Вопросов по техническому заданию
- Уточнения деталей у менеджера
- Сообщений о готовности этапов
- Координации с коллегами по проекту

**Кто видит чат проекта:**
- Все сотрудники с ролями «Менеджер», «Руководитель» и «Производство»

## 7. Календарь мероприятий

Раздел **«Календарь»** — годовой вид мероприятий в открытых месяцах.

**Как работает:**
- Сетка по неделям (Пн–Вс)
- Проекты отображаются цветными метками на датах мероприятий
- Каждый менеджер — свой цвет
- Наведение на метку — краткая информация

**Только открытые месяцы:**
- Проекты отображаются только в месяцах, открытых руководителем
- Закрытые месяцы визуально недоступны

**Быстрый переход к проекту:**
- Нажмите на цветную метку в календаре → откроется карточка проекта

**Используйте календарь для:**
- Планирования рабочей нагрузки
- Просмотра ближайших мероприятий
- Понимания очерёдности работ

## 8. Склад — инвентарь

Раздел **«Инвентарь»** — учёт реквизита и оборудования на складе.

### Структура инвентаря

Инвентарь организован в **дерево категорий**:
- Раскрывайте категории кнопкой «►»
- Внутри каждой категории — список позиций

Примеры категорий: Арки, Вазы, Подсвечники, Текстиль, Оборудование и т.д.

### Просмотр позиции инвентаря

Нажмите на позицию — откроется карточка:
- Название
- Количество на складе
- Фотографии
- История повреждений

### Обновление количества

Если вы меняете количество единиц на складе (взяли на мероприятие / вернули):

1. Откройте карточку позиции
2. Нажмите **«Редактировать»** (если доступно)
3. Измените количество
4. Сохраните

### Фиксация повреждения

Если реквизит был повреждён в ходе мероприятия:

1. Найдите позицию в инвентаре
2. Откройте карточку
3. Нажмите **«Зафиксировать повреждение»**
4. Заполните форму:
   - Описание повреждения (что именно сломано/испорчено)
   - Дата
   - Количество повреждённых единиц
5. Нажмите **«Сохранить»**

Запись о повреждении сохраняется в журнале и видна руководителю.

> **Важно**: фиксируйте повреждения сразу — это помогает поддерживать актуальный учёт склада и своевременно заменять реквизит.

### Поиск по инвентарю

Используйте поисковую строку вверху раздела для быстрого поиска нужной позиции по названию.

## 9. Склад — ткани

Раздел **«Ткани»** — отдельный учёт текстиля.

**Что здесь:**
- Список тканей: название, артикул, цвет, количество
- Поиск по названию/артикулу

**Работа с тканями аналогична инвентарю:**
- Просмотр наличия
- Обновление количества после использования на мероприятии

## 10. История изменений

Раздел **«История»** — журнал ваших действий в системе.

**Что фиксируется:**
- Какие задачи вы отмечали выполненными (с датой и временем)
- Изменения в инвентаре
- Другие действия в системе

**Используйте историю для:**
- Подтверждения выполненной работы
- Проверки дат завершения задач

---

## 11. Общий чат

Раздел **«Сообщения»** — общий командный чат для всей студии.

**Как пользоваться:**
1. Откройте раздел «Сообщения»
2. Введите сообщение
3. Нажмите **Enter** или кнопку отправки

**Чем отличается от чата проекта:**
- Общий чат — для всей команды, общие вопросы и объявления
- Чат проекта — только для обсуждения конкретного мероприятия

## Типичные рабочие сценарии

### Сценарий 1: Начало рабочего дня

1. Войти в систему
2. Открыть **Проекты** — посмотреть ближайшие мероприятия
3. Открыть ближайший проект → вкладка **Задачи** — уточнить что нужно сделать сегодня
4. При вопросах — написать в **Чат проекта**

### Сценарий 2: Выполнение производственных задач

1. Открыть **Проект**
2. Вкладка **Задачи**
3. Последовательно отмечать задачи по мере выполнения
4. При необходимости уточнить детали в **Чате проекта**

### Сценарий 3: Получение материалов со склада

1. Открыть **Инвентарь**
2. Найти нужные позиции
3. Проверить наличие на складе
4. После получения материалов — обновить количество

### Сценарий 4: Возврат реквизита после мероприятия

1. Открыть **Инвентарь**
2. Найти использованные позиции
3. Обновить количество (добавить возвращённые единицы)
4. Если что-то повреждено — **зафиксировать повреждение** с описанием

### Сценарий 5: Обнаружение проблемы

1. Написать в **Чат проекта** — менеджер и руководитель увидят сразу
2. Либо использовать **Общий чат** для срочных вопросов

## Частые вопросы

**Почему список проектов пуст?**
Руководитель ещё не открыл текущий месяц. Уточните у руководителя.

**Как отменить отметку выполненной задачи?**
Повторно нажмите на чекбокс задачи — отметка снимается.

**Как сообщить о повреждении реквизита?**
Инвентарь → найти позицию → открыть карточку → «Зафиксировать повреждение».

**Я не вижу нужный проект. Что делать?**
Проверьте, открыт ли месяц мероприятия. Если да — напишите руководителю или менеджеру.
