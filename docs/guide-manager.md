# Инструкция: Менеджер

Роль **Менеджер** отвечает за работу с клиентами: от первого обращения до подписания договора и передачи проекта в производство. Менеджер видит только **своих** клиентов, договоры и проекты — данные других менеджеров скрыты.

## 1. Вход в систему

1. Перейдите по адресу системы (например, `https://crm.maketdecor.ru`)
2. Введите логин и пароль
3. Нажмите **«Войти»**

В боковом меню слева отображаются доступные разделы. Разделы, недоступные вашей роли, скрыты.

---

## 2. Главная (Дашборд)

После входа вы попадаете на дашборд.

**Элементы:**
- Приветственный экран с фоновым изображением
- Уведомления (колокольчик в шапке справа) — оповещения о новых заявках, изменениях статусов
- Быстрый доступ к ключевым разделам

**Смена фона:**
1. Нажмите иконку смены обоев (шестерёнка / иконка изображения)
2. Выберите вариант из галереи

## 3. Клиенты — воронка продаж

Раздел **«Клиенты»** — основное рабочее место менеджера.

### Статусы воронки

```
MEETING → DISCUSSION → ESTIMATE → CONTRACT
                                ↘
                              REJECTED
```

| Статус | Когда использовать |
|--------|-------------------|
| `MEETING` | Первый контакт, встреча назначена |
| `DISCUSSION` | Встреча прошла, обсуждаем детали |
| `ESTIMATE` | Готовим/отправили смету |
| `CONTRACT` | Договор подписан |
| `REJECTED` | Клиент отказался |

### Создание нового клиента

1. Нажмите кнопку **«Добавить клиента»** (правый верхний угол)
2. Заполните форму:
   - **Имя клиента** — ФИО или название компании
   - **Телефон, Email** — контактные данные
   - **Дата обращения** — когда поступил запрос
   - **Дата встречи** — планируемое время встречи
   - **Дата мероприятия** — когда пройдёт событие
   - **Площадка** — место проведения
   - **Тип мероприятия**: Свадьба, Корпоратив, День рождения, Другое
   - **Описание идеи** — что хочет клиент
3. Нажмите **«Сохранить»**

Клиент появится в списке со статусом `MEETING`.

### Обновление статуса клиента

1. Откройте карточку клиента (нажмите на строку)
2. В блоке «Статус» выберите новый статус
3. Изменение сохраняется автоматически

### Просмотр списка клиентов

- Таблица с сортировкой по любой колонке
- Фильтры: по статусу, типу мероприятия, дате мероприятия
- Поиск по имени/телефону

### Конвертация клиента в договор

Когда клиент готов подписать договор:

1. Откройте карточку клиента (статус должен быть `ESTIMATE` или выше)
2. Нажмите **«Конвертировать в договор»**
3. Подтвердите действие

Что происходит автоматически:
- Создаётся запись в разделе «Договоры» со всеми данными клиента
- Прикреплённые сметы переносятся в договор
- Статус клиента меняется на `CONTRACT`

### Отклонение клиента

1. Откройте карточку клиента
2. Нажмите **«Отклонить»**
3. Введите причину отказа (обязательно — для статистики)
4. Нажмите **«Подтвердить»**

Клиент переходит в статус `REJECTED` и учитывается в аналитике воронки.

## 4. Работа со сметами

Сметы прикрепляются к клиенту и переносятся в договор при конвертации.

### Загрузка сметы

1. Откройте карточку клиента
2. Перейдите на вкладку **«Сметы»** / **«Файлы»**
3. Нажмите **«Загрузить файл»**
4. Выберите файл (Excel, PDF) с компьютера
5. Укажите **версию сметы** (1, 2, 3... — каждая новая редакция)
6. При необходимости — отметьте **«Командная версия»** (без цен, для показа команде)

### Версии смет

Система хранит несколько версий одной сметы:
- Версия 1, 2, 3... — последовательные редакции
- Каждая версия хранится отдельно
- Можно скачать любую версию

### Командная версия

Это копия сметы **без цен** — для передачи в производство или дизайнеру. Загружается отдельно с отметкой «командная».

## 5. Договоры

Раздел **«Договоры»** — договоры, созданные из ваших клиентов.

### Карточка договора

Содержит:
- Данные клиента
- **Дата мероприятия** и **дата подписания договора**
- **Сумма договора** и условия предоплаты
- **Номер счёта**
- **Статус макета** — отслеживание работы дизайнера:

| Статус макета | Значение |
|---------------|---------|
| `PENDING` | Ожидает начала работы |
| `IN_PROGRESS` | Дизайнер работает |
| `APPROVED` | Макет согласован с клиентом |
| `TRANSFERRED` | Передан в производство |
| `CANCELLED` | Отменён |

### Прикрепление файлов к договору

1. Откройте договор
2. Вкладка **«Файлы»**
3. Загрузите файл договора (Word/PDF) и/или счёт
4. Укажите тип файла при загрузке

### Референсные изображения

1. Откройте договор
2. Вкладка **«Изображения»** / **«Референсы»**
3. Загрузите фотографии, которые клиент прислал как вдохновение
4. Дизайнер сможет просмотреть их при создании макета

### Редактирование договора

1. Откройте договор → кнопка **«Редактировать»**
2. Внесите изменения
3. Нажмите **«Сохранить»**

## 6. Проекты

Раздел **«Проекты»** — производственные карточки мероприятий.

Проект обычно создаётся после подписания договора и передачи задания в производство.

### Создание проекта

1. Нажмите **«Создать проект»**
2. Заполните:
   - **Название** проекта
   - **Дата мероприятия**
   - **Площадка**
   - **Описание** — технические детали для производства
   - **Цвет** — цветовая метка в календаре (выберите любой)
3. Нажмите **«Сохранить»**

### Связь с договором

При создании проекта его можно связать с договором. Это позволяет:
- Видеть полную цепочку: клиент → договор → проект
- Менеджеру отслеживать производство по своему договору

## 7. Задачи и закупки проекта

### Производственные задачи

Задачи — чеклист того, что нужно сделать производству.

**Добавление задачи:**
1. Откройте проект
2. Вкладка **«Задачи»**
3. Нажмите **«Добавить задачу»**
4. Введите название задачи
5. Нажмите **«Сохранить»**

**Отслеживание выполнения:**
- Каждая задача отображается с чекбоксом
- Когда производство отмечает задачу выполненной — видно кто и когда
- Общий прогресс показывается как N/M задач

### Список закупок

Материалы и расходники, которые нужно приобрести для проекта.

**Добавление позиции:**
1. Вкладка **«Закупки»**
2. **«Добавить позицию»**
3. Введите название, количество, единицы измерения
4. Сохраните

**Отметка о покупке:**
- Позиция отмечается куплен/не куплен
- Виден статус готовности закупок в целом

## 8. Чат проекта

У каждого проекта есть внутренний чат для команды.

**Как пользоваться:**
1. Откройте проект
2. Вкладка **«Чат»**
3. Введите сообщение в поле внизу
4. Нажмите **Enter** или кнопку отправки

Сообщения доставляются в реальном времени всем участникам проекта (менеджер + руководитель + производство).

**Используйте чат проекта для:**
- Уточнения деталей с производством
- Координации по конкретному мероприятию
- Обсуждения нюансов оформления

## 9. Фотографии проекта

К проекту можно прикреплять изображения двух типов:

| Тип | Описание |
|-----|---------|
| Эскизы заказа | Рисунки, схемы, референсы от клиента |
| Фото производства | Фотографии готовых изделий, сборки, монтажа |

**Загрузка фото:**
1. Откройте проект → вкладка **«Фотографии»**
2. Выберите тип (эскизы / производство)
3. Нажмите область загрузки или перетащите файлы
4. Поддерживаются форматы: JPG, PNG, WebP

## 10. Календарь мероприятий

Раздел **«Календарь»** — годовой вид со всеми вашими мероприятиями.

**Как работает:**
- Сетка по неделям (Пн–Вс), по месяцам
- Проекты отображаются цветными метками в ячейке даты
- Каждый проект — своим цветом (выбирается при создании проекта)
- Наведение на метку — краткая информация о мероприятии

**Быстрый переход:**
- Нажмите на метку в календаре → откроется карточка проекта

---

## 11. Макеты дизайнера

Раздел **«Дизайнер»** — просмотр макетов для ваших проектов.

**Что можно посмотреть:**
- Загруженные дизайнером изображения по зонам (церемония, президиум, банкетный зал и т.д.)
- Статус готовности макета
- Дату монтажа и сложность

Менеджер в этом разделе работает **только в режиме просмотра** — создание и редактирование макетов выполняет дизайнер.

## 12. Статистика

Раздел **«Статистика»** — ваши личные показатели.

**Доступные данные:**

| Метрика | Описание |
|---------|---------|
| Воронка продаж | Сколько клиентов на каждом этапе |
| Конверсия | % клиентов, переходящих от встречи к договору |
| Выручка по месяцам | Суммы подписанных договоров |
| Количество мероприятий | По типам и по месяцам |

Данные отображаются только по вашим клиентам.

## 13. Личные задачи и расходы

### Личные задачи

Раздел **«Мои задачи»** — персональный таск-менеджер менеджера.

**Создание задачи:**
1. **«Добавить задачу»**
2. Введите описание, срок выполнения
3. Сохраните

**Работа с задачами:**
- Отметьте выполненную задачу — она переходит в архив
- Сортировка по сроку, приоритету
- Задачи видны только вам

### Расходы

Учёт расходов, понесённых по работе.

**Добавление расхода:**
1. **«Добавить расход»**
2. Заполните: сумма, описание, дата, тип расхода
3. При необходимости — привяжите к конкретному проекту
4. Сохраните

## 14. Входящие заявки с сайта

Раздел **«Входящие»** — заявки с формы на сайте студии.

**Что здесь:**
- Список заявок: имя, контакт, текст, дата
- Заявки создаются автоматически при получении email

**Обработка заявки:**
1. Изучите заявку
2. Нажмите **«Создать клиента»** — данные из заявки перенесутся в форму нового клиента
3. Дополните информацию и сохраните клиента

Раздел **«История»** — журнал ваших действий в системе.

Здесь видны все изменения, которые вы вносили:

- Смена статусов

## 16. Общий чат

Раздел **«Сообщения»** — общий командный чат.

- Все сотрудники видят одну ленту сообщений
- Сообщения приходят в реальном времени
- Используйте для общих объявлений и координации

> Для обсуждения конкретного проекта используйте **чат проекта** (раздел «Проекты» → карточка проекта → вкладка «Чат»).

## Пример пути пользователя в платформе (CJM)

Ниже — подробный сквозной пример действий менеджера Анны после авторизации. Он показывает, как разделы платформы соединяются в единый рабочий процесс — от входа в систему до сдачи мероприятия.

### Шаг 0. Авторизация
1. Анна открывает `https://crm.maketdecor.ru`, вводит логин и пароль, нажимает **«Войти»**.
2. Система перенаправляет её на **Дашборд**. В шапке справа — колокольчик с уведомлениями (3 новых).
3. В боковом меню видны только доступные роли разделы: Дашборд, Клиенты, Договоры, Проекты, Календарь, Дизайнер, Входящие, Сообщения, Мои задачи, Расходы, Статистика, История.

### Шаг 1. Проверка уведомлений и входящих заявок
1. Анна кликает по **колокольчику** — видит уведомление «Новая заявка с сайта».
2. Переходит в **«Входящие»** → видит заявку: «Ольга, свадьба, 12 июля, +7 999 ...».
3. Открывает заявку, читает текст, нажимает **«Создать клиента»**.
4. В открывшейся форме данные заявки уже подставлены. Анна дополняет:
   - Тип мероприятия — **Свадьба**
   - Дата мероприятия — **12.07.2026**
   - Дата встречи — **10.04.2026, 15:00**
   - Описание идеи — «бохо, пастельные тона, выездная регистрация»
5. Нажимает **«Сохранить»** — клиент Ольга появляется в воронке со статусом `MEETING`.

### Шаг 2. Подготовка ко встрече
1. Анна заходит в **«Клиенты»**, открывает карточку Ольги.
2. На вкладке **«Файлы»** загружает PDF с примерами оформления (моудборд) — пригодится на встрече.
3. Открывает **«Мои задачи»** → **«Добавить задачу»**: «Подготовить прайс для Ольги», срок — 09.04.
4. После встречи открывает карточку клиента и меняет статус с `MEETING` на `DISCUSSION`.

### Шаг 3. Работа со сметой
1. На вкладке **«Сметы»** карточки клиента нажимает **«Загрузить файл»** → выбирает Excel-смету, версия 1.
2. После замечаний клиента Анна правит смету у себя на компьютере и загружает версию 2.
3. Дополнительно загружает **«командную версию»** без цен — для дизайнера и производства.
4. Меняет статус клиента на `ESTIMATE`.

### Шаг 4. Конвертация в договор
1. Клиент согласовал смету. Анна в карточке клиента нажимает **«Конвертировать в договор»** → подтверждает.
2. Система автоматически создаёт запись в **«Договоры»**, переносит сметы, статус клиента становится `CONTRACT`.
3. Анна переходит в **«Договоры»**, открывает новый договор Ольги:
   - Заполняет **сумму договора**, условия предоплаты, **номер счёта**, **дату подписания**.
   - На вкладке **«Файлы»** загружает подписанный PDF договора и счёт.
   - На вкладке **«Референсы»** загружает 8 фото-вдохновений от клиента — для дизайнера.
4. Сохраняет изменения. Статус макета остаётся `PENDING` — ждёт дизайнера.

### Шаг 5. Передача в дизайн и производство
1. Анна открывает **«Сообщения»** (общий чат) и пишет: «Ольга, свадьба 12.07 — договор подписан, референсы залиты, можно брать в работу @дизайнер».
2. Переходит в **«Проекты»** → **«Создать проект»**:
   - Название: «Свадьба Ольги, 12.07»
   - Дата мероприятия, площадка, описание зон
   - Цвет метки в календаре — пастельно-розовый
   - Привязывает проект к договору Ольги.
3. На вкладке **«Задачи»** добавляет производственные позиции: «Сборка арки», «Декор президиума», «Подсветка зала».
4. На вкладке **«Закупки»** добавляет: «Цветы — пионы 60 шт.», «Свечи 30 шт.», «Ткань шифон 20 м».
5. На вкладке **«Фотографии»** → **«Эскизы заказа»** загружает референсы.
6. Открывает **«Чат проекта»** и пишет производству: «Монтаж 12.07 с 8:00, доступ через служебный вход».

### Шаг 6. Контроль работы дизайнера
1. Через 2 дня Анна заходит в раздел **«Дизайнер»**, фильтрует по своему проекту.
2. Видит: статус макета `IN_PROGRESS`, загружены изображения зоны «церемония» и «банкетный зал».
3. Скачивает картинки, отправляет клиенту в мессенджере, получает правки.
4. Возвращается в **общий чат** и пишет дизайнеру список правок.
5. После согласования статус макета меняется на `APPROVED` — Анна видит это в карточке договора.

### Шаг 7. Подготовка к мероприятию
1. За неделю до события Анна заходит в **«Календарь»**, находит свою цветную метку 12.07, открывает проект.
2. Проверяет прогресс **задач** (например, 4/6 выполнено) и **закупок** (8/10 куплено).
3. В **чате проекта** уточняет у производства, когда докупят оставшиеся материалы.
4. На вкладке **«Фотографии» → «Производство»** видит фото готовой арки — пересылает клиенту.

### Шаг 8. После мероприятия
1. 13.07 Анна открывает **«Проекты»** → проект Ольги → отмечает все задачи завершёнными (если ещё не закрыты).
2. Загружает финальные фото с площадки в **«Фотографии производства»**.
3. Заходит в **«Расходы»**, добавляет траты, понесённые лично (такси на встречу, распечатка договора), привязывает к проекту.
4. Открывает **«Статистика»** — проверяет, как обновилась её воронка и выручка за июль.
5. В **«Истории изменений»** при необходимости смотрит хронологию работы с клиентом.

> Этот сквозной путь — типовой день/цикл менеджера. Все разделы платформы участвуют в едином потоке: **Входящие → Клиенты → Сметы → Договоры → Проекты → Дизайнер → Производство → Статистика**.

## Типичные рабочие сценарии

### Сценарий 1: Обработка новой заявки с сайта

1. Раздел **«Входящие»** → найти новую заявку
2. Нажать **«Создать клиента»**
3. Дополнить контакты, поставить статус `MEETING`, назначить дату встречи
4. Сохранить

### Сценарий 2: Ведение клиента от встречи до договора

1. **Создать клиента** со статусом `MEETING`
2. После встречи — обновить статус → `DISCUSSION`
3. Подготовить смету → **загрузить файл** → статус → `ESTIMATE`
4. Клиент согласен → **«Конвертировать в договор»**
5. В договоре заполнить сумму, даты, загрузить файл договора

### Сценарий 3: Подготовка проекта для производства

1. Создать **проект** по договору
2. Добавить **задачи производства** — что нужно сделать
3. Добавить **список закупок** — что нужно купить
4. Загрузить **фото эскизов** для ориентира
5. Написать в **чат проекта** — уведомить команду

### Сценарий 4: Мониторинг готовности к мероприятию

1. Открыть **проект**
2. Проверить прогресс задач (N/M выполнено)
3. Уточнить детали в **чате проекта**
4. Проверить статус **закупок**
